Notes: Dino Sprite Animation in CSS

1. Original CSS

.dino-corriendo { animation: animarDino 0.25s steps(2) infinite; } @keyframes animarDino { from { background-position-x: -84px; } to { background-position-x: -252px; } }

2. What does this code do?

This code makes the dinosaur look like it is running by changing which part of a larger background image is shown.

The dinosaur image is a sprite sheet, which means several animation frames are stored inside one single image.

3. What is a sprite sheet?

A sprite sheet is one image that contains many small pictures placed side by side. Each small picture is called a frame.

| frame 1 | frame 2 | frame 3 | frame 4 | 0--------84--------168-------252------336

In your case:

  • Each frame is 84px wide
  • The full image width is 336px
  • So there are 4 frames total

4. Why is background-position-x negative?

The dinosaur box is only 84px wide, so it can show only one frame at a time. The full sprite image is much wider, so CSS moves the large image left or right behind this small box.

Important idea:
The box does not move.
The background image moves inside the box.

When the value is negative, the background image moves to the left. That makes the next frame appear inside the visible box.

5. Frame positions

background-position-x What happens Visible frame
0px The image starts at the left edge Frame 1
-84px The image moves 84px to the left Frame 2
-168px The image moves 168px to the left Frame 3
-252px The image moves 252px to the left Frame 4

6. Why does moving left show the next frame?

Imagine you have a small window, and behind it there is a long strip of pictures.

Small visible box: |---------84px---------| Full sprite image: | frame 1 | frame 2 | frame 3 | frame 4 |

If the image is at 0px, you see frame 1.

background-position-x: 0px Box: |---------| Image: | frame 1 | frame 2 | frame 3 | frame 4 | ^ image starts here, so box shows frame 1

If the image moves left by 84px, frame 1 goes out of view, and frame 2 moves into the box.

background-position-x: -84px Box: |---------| Image: | frame 1 | frame 2 | frame 3 | frame 4 | ^ image shifted left, so box now shows frame 2

So the values are negative because we want to shift the full image to the left.

7. Why not positive values?

Positive values move the background image to the right. That usually shows the wrong part or creates empty space.

Since the sprite frames are arranged from left to right, to see later frames we move the full image left, so we use negative values.

8. What does this animation mean?

animation: animarDino 0.25s steps(2) infinite;
  • animarDino → the name of the animation
  • 0.25s → the animation lasts 0.25 seconds
  • steps(2) → changes happen in jumps, not smoothly
  • infinite → it repeats forever
For sprite animation, steps() is useful because we want to jump from one frame to another, not slide smoothly.

9. Why does the animation start at -84px instead of 0px?

Usually 0px is used for the dinosaur's standing frame. The running animation may use only the next frames, so it starts at -84px.

That means:

  • 0px → standing frame
  • -84px, -168px, -252px → running frames

10. Simple summary

  • The dinosaur image contains several frames in one long image.
  • The visible box is only 84px wide.
  • Negative background-position-x moves the full image left.
  • Moving left reveals the next frame.
  • That is why frame 2, 3, and 4 use negative values.

11. Short memory trick

Negative x = move background left = show a later frame
