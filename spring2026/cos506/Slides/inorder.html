Step-by-Step Explanation of Inorder Traversal

Code

void inorder(Node* root) {
    if (root == nullptr) return;
    inorder(root->left);
    std::cout << root->key << " ";
    inorder(root->right);
}

Main Idea

Inorder traversal visits nodes in this order: Left → Root → Right.

That means for each node:

  • Go to the left child first
  • Print the current node
  • Go to the right child

Example Tree

10 / \ 5 15 / \ 12 20

Step-by-Step Trace

Step 1: Start at node 10

Call: inorder(10)

For node 10, the function does:

  • inorder(10->left) → inorder(5)
  • Print 10
  • inorder(10->right) → inorder(15)

Step 2: Move to the left of 10 → node 5

Call: inorder(5)

For node 5:

  • inorder(5->left) → nullptr, so return
  • Print 5
  • inorder(5->right) → nullptr, so return

Output so far: 5

Step 3: Return to node 10

After finishing the left subtree, print 10.

Output so far: 5 10

Step 4: Move to the right of 10 → node 15

Call: inorder(15)

For node 15:

  • inorder(15->left) → inorder(12)
  • Print 15
  • inorder(15->right) → inorder(20)

Step 5: Move to the left of 15 → node 12

Call: inorder(12)

  • inorder(12->left) → nullptr, so return
  • Print 12
  • inorder(12->right) → nullptr, so return

Output so far: 5 10 12

Step 6: Return to node 15

Now print 15.

Output so far: 5 10 12 15

Step 7: Move to the right of 15 → node 20

Call: inorder(20)

  • inorder(20->left) → nullptr, so return
  • Print 20
  • inorder(20->right) → nullptr, so return

Output so far: 5 10 12 15 20

Call Flow

inorder(10)
 ├─ inorder(5)
 │   ├─ inorder(nullptr)
 │   ├─ print 5
 │   └─ inorder(nullptr)
 ├─ print 10
 └─ inorder(15)
     ├─ inorder(12)
     │   ├─ inorder(nullptr)
     │   ├─ print 12
     │   └─ inorder(nullptr)
     ├─ print 15
     └─ inorder(20)
         ├─ inorder(nullptr)
         ├─ print 20
         └─ inorder(nullptr)
Final inorder output: 5 10 12 15 20

Simple Summary

Inorder traversal always follows Left → Root → Right. That is why for this tree, the nodes are printed in the order: 5 10 12 15 20.
