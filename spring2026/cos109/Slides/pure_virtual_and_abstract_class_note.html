Note: Pure Virtual Function and Abstract Class in C++

1. Pure Virtual Function

A pure virtual function is a function declared in a base class that has no implementation in that class.

class Vehicle {
public:
    virtual void stop() = 0;
};

In this example, stop() is a pure virtual function.

What does this mean?

  • virtual tells C++ to use runtime polymorphism.
  • = 0 means the function is pure virtual.
  • The base class does not provide the real behavior for this function.
  • Any derived class must write its own version of stop().
A pure virtual function says: "Every child class must define this function."

Example with derived classes

class Car : public Vehicle {
public:
    void stop() override {
        cout << "Car stops by pressing the brake pedal." << endl;
    }
};

class Bicycle : public Vehicle {
public:
    void stop() override {
        cout << "Bicycle stops by squeezing the hand brakes." << endl;
    }
};

Here, both Car and Bicycle provide their own implementation of stop().

2. Abstract Class

An abstract class is a class that cannot be used to create objects directly. It is meant to be a parent class for other classes.

class Vehicle {
public:
    virtual void stop() = 0;
};

Because Vehicle contains a pure virtual function, it becomes an abstract class.

What does abstract mean here?

It means Vehicle is only a general idea, not a specific object.

  • Vehicle is a general category.
  • Car, Bicycle, and Truck are specific types of vehicles.

Allowed and not allowed

Vehicle v;   // Not allowed
Car car;         // Allowed
Bicycle bike;    // Allowed
Truck truck;     // Allowed
An abstract class is used to provide a common structure for child classes.

3. Relationship Between Them

A class becomes abstract when it has at least one pure virtual function.

virtual void stop() = 0;

So in this project:

  • stop() is the pure virtual function.
  • Vehicle is the abstract class.

4. Simple Real-Life Idea

Think of Vehicle as a general idea. A vehicle must stop, but different vehicles stop in different ways.

  • A car stops with a brake pedal.
  • A bicycle stops with hand brakes.
  • A truck may stop with air brakes.

So the parent class gives the rule, and the child classes give the details.

5. Quick Summary

  • A pure virtual function is written like virtual void stop() = 0;
  • It forces child classes to implement that function.
  • A class with a pure virtual function becomes an abstract class.
  • An abstract class cannot create objects directly.
  • It is used as a base class for derived classes.
