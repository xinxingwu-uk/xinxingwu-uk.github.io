C++ OOP Concepts

1. Simple Meanings

Overloading

Overloading means using the same function name with different parameter lists.

Overriding

Overriding means a derived class provides a new version of a base class virtual function.

Overwrite

Overwrite usually means replacing an old value with a new value. It is not a special OOP feature like overloading or overriding.

Polymorphism

Polymorphism means one base-class pointer can point to different child objects, and the correct function runs automatically.

2. Complete Example

#include <iostream>
#include <string>
using namespace std;

class Person {
public:
    string name;
    int age;

    Person(string n, int a) {
        name = n;
        age = a;
    }

    // Overloaded function #1
    void setInfo(string n) {
        name = n;
    }

    // Overloaded function #2
    void setInfo(string n, int a) {
        name = n;
        age = a;
    }

    // Virtual function
    virtual void showInfo() {
        cout << "Person Information:" << endl;
        cout << "Name: " << name << endl;
        cout << "Age: " << age << endl;
    }
};

class Student : public Person {
public:
    string studentID;

    Student(string n, int a, string id) : Person(n, a) {
        studentID = id;
    }

    // Override
    void showInfo() override {
        cout << "Student Information:" << endl;
        cout << "Name: " << name << endl;
        cout << "Age: " << age << endl;
        cout << "Student ID: " << studentID << endl;
    }
};

class Teacher : public Person {
public:
    string subject;

    Teacher(string n, int a, string s) : Person(n, a) {
        subject = s;
    }

    // Override
    void showInfo() override {
        cout << "Teacher Information:" << endl;
        cout << "Name: " << name << endl;
        cout << "Age: " << age << endl;
        cout << "Subject: " << subject << endl;
    }
};

int main() {
    Student s("Alice", 20, "S101");

    // Overloading
    s.setInfo("Alice Johnson");
    s.setInfo("Alice Johnson", 21);

    // Overwrite old value with new value
    s.studentID = "S202";

    s.showInfo();

    cout << endl;

    // Polymorphism
    Person* p1 = new Person("Tom", 40);
    Person* p2 = new Student("Bob", 19, "S303");
    Person* p3 = new Teacher("Mr. Brown", 35, "Math");

    p1->showInfo();
    cout << endl;

    p2->showInfo();
    cout << endl;

    p3->showInfo();
    cout << endl;

    delete p1;
    delete p2;
    delete p3;

    return 0;
}

3. Where Each Concept Appears

Overloading

void setInfo(string n)
void setInfo(string n, int a)

These functions have the same name, but different parameters.

Overriding

virtual void showInfo()
void showInfo() override

The child classes Student and Teacher provide their own versions of showInfo().

Overwrite

s.studentID = "S202";

This replaces the old value "S101" with the new value "S202".

Polymorphism

Person* p2 = new Student("Bob", 19, "S303");
Person* p3 = new Teacher("Mr. Brown", 35, "Math");

Even though the pointers are of type Person*, the correct child version of showInfo() runs.

4. Important Note

If you remove virtual from Person::showInfo(), then polymorphism will not work correctly. In that case, a Person* pointer would call the base class version instead of the child class version.

5. Comparison Table

7. Short Summary

Overloading = same name, different inputs

Overriding = child changes parent function

Overwrite = replace old value with new value

Polymorphism = one pointer, many forms
