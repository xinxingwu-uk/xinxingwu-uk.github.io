Object Slicing vs Polymorphism in C++

Why is the output different?

The two versions of code produce different output because they work in different ways.

1. Object Version

Person per = Person("John", 30);
Person stu = Students("Oscar", 20, "12334");
Person teacher = Teachers("JaSun", 21, "COS201");

In this version, stu and teacher are declared as Person objects.

Even though the right side creates a Students or Teachers object, only the Person part is kept.

The extra derived-class data, such as StudentID or subject, is removed.

This is called object slicing.

So later: 

stu.showInfo();
teacher.showInfo();

both call Person::showInfo(), because both are now only Person objects.

2. Pointer Version

Person* per = new Person("John", 30);
Person* stu = new Students("Oscar", 20, "12334");
Person* teacher = new Teachers("JaSun", 21, "COS201");

In this version, stu and teacher are pointers of type Person*, but they point to real Students and Teachers objects.

Since showInfo() is declared as virtual in the base class, C++ uses runtime polymorphism. 

stu->showInfo();
teacher->showInfo();

So:

  • stu->showInfo() calls Students::showInfo()
  • teacher->showInfo() calls Teachers::showInfo()

Main Difference

Object version: causes object slicing

Pointer version: keeps the full derived object and allows polymorphism

Simple Summary

When you write: 

Person stu = Students(...);

only the Person part is stored.

But when you write: 

Person* stu = new Students(...);

the whole Students object is still there, and the virtual function works correctly.

Expected Output

Object version:

Person info:
Name: John
Age: 30
--------------------------------
Person info:
Name: Oscar
Age: 20
--------------------------------
Person info:
Name: JaSun
Age: 21

Pointer version:

Name: John
Age: 30

Age: 20

Conclusion

The output is different because the first version uses object slicing, while the second version uses polymorphism with virtual functions.
