C++ Notes: Class, Object, and Instance

An explanation based on MyClass

Original Code

class MyClass {       // The class
  public:             // Access specifier
    int myNum;        // Attribute (int variable)
    string myString;  // Attribute (string variable)
};

1. Class

A class is like a blueprint or template.

It tells C++ what kind of data something should have.

class MyClass {
  public:
    int myNum;
    string myString;
};

MyClass is the class.

It says: “Anything made from this class will have:”

  • one integer: myNum
  • one string: myString

2. Object

An object is a real thing created from the class.

Example

MyClass obj1;

Now obj1 is an object of class MyClass.

You can put values into it:

obj1.myNum = 10;
obj1.myString = "Hello";

So obj1 now stores real data.

3. Instance

An instance means the same thing as an object in most beginner C++ explanations.

MyClass obj1;

obj1 is:

  • an object
  • an instance of MyClass

Full Example

#include <iostream>
#include <string>
using namespace std;

class MyClass {       
  public:             
    int myNum;        
    string myString;  
};

int main() {
    MyClass obj1;   // obj1 is an object (instance) of MyClass
    MyClass obj2;   // obj2 is another object (instance)

    obj1.myNum = 10;
    obj1.myString = "Apple";

    obj2.myNum = 20;
    obj2.myString = "Banana";

    cout << obj1.myNum << " " << obj1.myString << endl;
    cout << obj2.myNum << " " << obj2.myString << endl;

    return 0;
}

Output

10 Apple
20 Banana

Real-Life Example

Think about a class like a house blueprint.

Programming Term Real-Life Meaning
Class Blueprint of a house
Object / Instance An actual house built from that blueprint

If you build two houses from the same blueprint:

MyClass obj1;
MyClass obj2;

Then obj1 and obj2 both follow the same design, but they can hold different values.

Class vs. Struct

class and struct are very similar in C++, but they are not exactly the same.

Main Difference

Keyword Default Access
class private
struct public

Example with class

class MyClass {
    int myNum;
    string myString;
};

Here, myNum and myString are private by default.

So this would cause an error:

MyClass obj;
obj.myNum = 10;   // error

That is why our earlier example used public:

class MyClass {
  public:
    int myNum;
    string myString;
};

Example with struct

struct MyStruct {
    int myNum;
    string myString;
};

Here, the members are public by default, so this works:

MyStruct obj;
obj.myNum = 10;
obj.myString = "Hello";

Important Idea

So the code:

class MyClass {
  public:
    int myNum;
    string myString;
};

is very similar to:

struct MyStruct {
    int myNum;
    string myString;
};

because both make the data members accessible from outside.

Easy rule to remember:
class defaults to private, while struct defaults to public.

In Code

  • MyClass → class
  • obj1, obj2 → objects
  • obj1, obj2 are also instances of MyClass
Note:
Class defines the architecture, object is the actual thing created from it, and instance is another word for that object.
